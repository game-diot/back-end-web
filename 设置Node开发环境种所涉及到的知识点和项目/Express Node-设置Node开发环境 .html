设置Node开发环境

Express开发环境是什么：

  • 完整的Express本地开发环境包括Node.js、NPM包管理器和Express应用生成器
  • Node和NPM包管理器可以用二进制包、安装程序或系统管理包一并安装。
    • 在开发每个Express web应用时，由NPM针对当前应用将Express（以及模板引擎、数据库驱动程序、身份验证中间件、静态文件托管中间件等其他库）作为依赖项进行安装
  • NPM也可以安装全局的Express应用生成器，可用于创建遵循MVC模式的Express应用框架。他不是必须的，因为无需这个工具就可以创建Express应用（使用相同的架构布局或依赖的Express应用）。他更容易上手，有助于应用结构的模块化管理
    • 典型的开发环境中还需要一些外部工具，包裹用于编写代码的文本编辑器，用于代码控制管理的工具。

设置Node开发环境

【安装Node】：

  • 现在系统上安装Nodejs和NPM之后才可以使用Express

【测试Nodejs和NPM是否安装成功】：

  • 终端种运行version命令，查看版本号
    • node/npm -v/version

【测试：构建纯Node服务器】：

  • hellonodejs文件中，编写Node特性与ES语法
    // 加载 HTTP 模块
    const http = require("http");
    const hostname = "127.0.0.1";
    const port = 3000;

    // 创建 HTTP 服务器
    const server = http.createServer((req, res) => {
    // 设置 HTTP 响应头：状态码和内容类型
    res.statusCode = 200;
    res.setHeader("Content-Type", "text/plain");

    // 发送响应体
    res.end("Hello World\n");
    });

    // 启动服务器监听指定端口
    server.listen(port, hostname, () => {
    console.log(
    服务器运行于 &lt;http://$&gt;{hostname}:${port}/);
    });

    • 代码导入了http模块，并使用它创建了一个服务器来监听端口的HTTP请求。随后在控制台打印一条信息，提示测试服务器的正确URL。
    • createServer（）函数接受一个回调函数作为参数，并在接收到HTTP请求后进行回调，直接返回HTTP状态码，以及纯文本信息
  • 命令行工具中进入hellonodejs文件夹，输入node + 文件名，运行，启动服务器
    • node hellonodejs
    • 服务器启动后，查看控制台输出，服务器运行的IP地址
  • 浏览器中访问URL ip地址。

【使用NPM】：

  • 构建Node应用过程中，NPM是除了Node本身之外最重要的工具，用于获取应用开发、测试以及生产所需要的所有包（JS库）。也可以运行开发过程中使用的测试单元和工具
  • 可以使用NPM手动安装图所需的包，在package.json文件中管理依赖，把每个依赖以JS包的形式罗列在文件中（包括名称、版本、描述、初始执行文件、生产依赖、开发依赖、支持的Node版本等等），package.json文件包含NPM获取和运行应用程序所需的所有内容（在编写可以重复使用的库时，可以把包上传到NPM仓库供其他用户使用）

【添加依赖项】：

  • 使用NPM下载包、将包保存进工程依赖书，以及Node应用中调用包的方法与步骤。
    • 为新应用创建目录，并进入
    • 使用NPM的init命令为应用创建package.json文件。这个命令将请求一系列的信息，包括应用的名称和版本，程序初始进入点的文件名（默认index.js）
      • npm init
        • package.json文件中保存了接受的默认信息，最后一条为许可证信息
    • 在myapp目录种安装Express，将Express保存在package.json文件中的依赖表里
      • npm install express
        • package.jso文件的底部会出现依赖列表（dependency），其中包含Express
    • 可以使用require（）函数来使用库，index.js文件
      const express = require("express");
      const app = express();
      const port = 3000;

      app.get("/", (req, res) => {
      res.send("Hello World!");
      });

      app.listen(port, () => {
      console.log(
      示例程序正在监听 ${port} 端口！);
      });

      • 代码展示了最简单的Express应用，导入了express模块并使用它创建类一个服务器app来监听3000端口，并且在控制台打印了一条信息来测试服务器的正确URL，
      • app.get（）函数只响应对特定路径（‘/’）的HTTP GET请求，此处响应就是发送helloworld
    • 在命令行种使用脚本调用node来启动服务器
    • 浏览器中访问URL

【开发依赖】：

  • 如果一个依赖只在开发过程中使用，应该将其保存为开发以来（这样，用户无需再生产环境种安装他们），比如，要使用eslint可以调用NPM
    • npm install eslint —save-dev
  • 当前的package.json文件中会自动添加以下项目
    • “devDependencies”：{
      • “eslint：“^版本””
    • }

【运行任务】

  • 在package.json种，除了定义和获取依赖，也可以定义脚本，然后通过NPM的 run-script命令来运行，这个用法普遍用于自动运行测试单元或部分应用，也可以用于构建工具链（比如运行工具来压缩JS文件或图片，init或分析代码，等等）
    • 比如在package.json文件夹中添加以下内容来定义脚本，从而对上文的代码运行eslint
      • "scripts": {
        ...
        "lint": "eslint src/js"
        ...
        }
        • eslint src/js命令可以把终端/命令行对应用目录下的src/js目录中的JS文件运行eslint。把上述脚本添加进应用的package.json种还可以为此命令添加一个快捷方式-lint
    • 之后就可以使用npm来运行eslint
      • npm run-script lint
        • 或者
      • npm run lint
  • 可以在NPM脚本中加入更长的命令，多条命令链，可以让单一的NPM脚本来一次运行所有的测试单元

【安装Express应用生成器】：

  • Express应用生成器构建可以生成一个Express应用的框架，可以使用NPM安装 -g全局安装
    • npm install express-generator -g
  • 进入应用目录，运行以下命令，即可创建一个名为helloworld的Express
    • express helloworld
  • NPM在当前位置的子目录创建新的Express应用，可以在控制台查看到构建的过程，完成后NPM会提示需要安装哪些Node依赖，以及如何开启应用
    • 进入目录安装依赖
      • npm install
    • 运行应用
      • SET DEBUG=helloworld:* & npm start
    • Debug命令可以展示应用运行时返回的有用的日志信息
    • 打开IP可以看到Express的默认欢迎页面
